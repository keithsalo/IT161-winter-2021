Accessibility

Accessibility is defined by Webster's Dictionary as “capable within reach”. When we apply this concept to the web it means that everyone should be able to use it regardless of any limitations or disabilities. The web has become an essential part of our society so it is imperative that even people with disabilities are able to utilize it.

There are a variety of ways that web accessibility can benefit those with disabilities. For the people who are hearing impaired this can consist of adding text to your code for visual content. Another way is to add sign language videos to the page to help descriptions or to disseminate live content. For people with visual impairments making the web more accessible could be using bigger size text and making sure the contrast is ok so the user can distinguish different elements on the page (W3C). Making the web more accessible is not only helpful for those with disabilities but also has similar benefits for the elderly, who might have cognitive, sight, or hearing decline.

Guidelines and framework for developers to use to keep their website more accessible is provided by the world wide web consortium or W3C. The W3C understood the need to make standards for people with disabilities and in 1997 the Web Accessibility Initiative or WAI was launched. The W3C is internationally recognized for their web standards and the WAI “ensures that W3C standards support accessibility” (W3C).

The future of web accessibility is as crazy and unpredictable as the web itself. According to the Bureau of Internet Accessibility “the disability rates in the United States hover around 25 percent, and if anything is likely to increase over the next decade” (BOIA). With technologies like wearable technology and voice assistance becoming more advanced they could play a bigger and more useful to people with disabilities moving forward. There are so many different paths for innovation and development it will be exciting to see what will be available.
